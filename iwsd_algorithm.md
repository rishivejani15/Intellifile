# Information Weighted Semantic Diff (IWSD) Algorithm

The IWSD algorithm is a proprietary mathematical risk engine designed for IntelliFile. Unlike standard Git-based algorithms that only count *how many* lines changed, IWSD calculates *how important* the changed data was using Shannon Entropy and Asymmetric Information Loss heuristics.

Here is the exact step-by-step logic of how the final algorithm operates.

---

### Step 1: Base Importance Calculation
Before scoring anything, the engine must understand how "dense" a single line of text is. When a line is passed to `get_importance()`, it calculates:
1. **Token Score:** How many words exist.
2. **Symbol Score:** How many non-alphanumeric characters exist (brackets, commas, quotes).
3. **Shannon Entropy:** A mathematical logarithmic measurement of character unpredictability.
The combination of these metrics gives the line a **Base Importance Score**. *(Note: This score is dynamically generated by the math formula in real-time, not statically assigned. For example, a blank space dynamically calculates to ~0.0, while a complex formula naturally calculates to a high entropy like ~4.5).*

---

### Step 2: The Scoring Engines
The engine separates the changes into three distinct engines based on the action taken:

#### A. The Addition Engine
Adding data is standard and low-risk.
* The engine loops through all newly added lines.
* It calculates the Base Importance Score of the line.
* It applies a standard **1.0x Weight Multiplier**.

#### B. The Deletion Engine (Asymmetric Scaling)
Destroying data is strictly riskier than adding data. 
* The engine loops through all removed lines and checks them against a **Heuristic Security Dictionary** and **Regex Patterns** (Credit Cards, SSNs, API Keys).
* **If Sensitive:** It bypasses the math and applies a massive **10.0x Penalty Multiplier**.
* **If Normal:** It checks the Shannon Entropy of the deleted text:
  * Entropy > 3.5 (Code/Math): **3.0x Multiplier**
  * Entropy > 2.0 (Sentences): **2.5x Multiplier**
  * Entropy < 2.0 (Basic Data): **2.0x Multiplier**

#### C. The Modification Engine (Information Downgrade Penalty)
A modification happens when an old line is replaced by a new line.
* The engine calculates the importance of the Old Line and the New Line.
* **If Old > New (Information Downgrade):** The user replaced complex data with simple data. The engine applies a **2.0x Risk Penalty** to the exact mathematical difference that was lost, heavily raising the risk score.
* **If New > Old (Information Upgrade):** The user safely added complexity. It applies a standard 1.0x weight.

---

### Step 3: Intent & Severity Tagging
1. **Intent Extraction:** The engine compares the final scores.
   - If the 10.0x Security Alarm tripped -> **"Sensitive Data Deletion"**
   - Else if `delete_score` > `add_score` -> **"Deletion Heavy"**
   - Else -> **"Addition/Modification"**
2. **Severity Aggregation:** The engine adds the Add, Delete, and Modify scores into a final `change_score`. It compares this mathematically against the total size of the original document. If the `change_score` crosses a massive threshold (due to the 3.0x or 10.0x penalties), the engine permanently locks the snapshot as a **"Major Risk"**.

---

## Real-World Example

Let's look at how the algorithm handles a real scenario inside a `config.py` file.

**The Original File:**
```python
API_KEY = "sk-live-1837492837482"
MAX_RETRIES = 5
DEBUG_MODE = True
```

**The User's Edits:**
1. The user deletes line 1 (`API_KEY`).
2. The user modifies line 2 to `MAX_RETRIES = 10`.
3. The user adds a new line: `# End of file`.

### How IWSD Processes This:

**1. The Addition:**
* Line: `# End of file`
* Math: Low Shannon Entropy. Base Score = 2.0.
* Multiplier: 1.0x
* **Add Score = 2.0**

**2. The Modification:**
* Old Line: `MAX_RETRIES = 5` (Base Score = 3.0)
* New Line: `MAX_RETRIES = 10` (Base Score = 3.1)
* Math: Since the New Line is more complex than the Old Line, there is no Information Downgrade penalty.
* **Modify Score = 3.1**

**3. The Deletion:**
* Line: `API_KEY = "sk-live-1837492837482"`
* Math: The engine scans the text and detects the exact keyword `"api_key"`. The Heuristic Security alarm trips!
* Base Score: 5.5
* Multiplier: 10.0x (Sensitive Data Penalty)
* **Delete Score = 55.0**

### The Final Result:
* **Total Change Score:** 2.0 + 3.1 + 55.0 = **60.1**
* Because `55.0` heavily outweighs `2.0`, the system tags the intent as **"Sensitive Data Deletion"**.
* Because `60.1` is massively disproportionate to the tiny 3-line file, the system immediately tags the edit as a **"Major Risk"**, attaching a Red Badge in the UI to warn the user!
